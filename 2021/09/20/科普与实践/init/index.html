hexo + githubActions 搭建的个人博客

2021-09-20- 2021-09-25
1.1k- 4m
科普与实践

GitHub，Gitee，Coding，静态博客「Pages」服务横评

本博客实现的工作流：

  1. 在本地某磁盘某文件夹下编写 .md 博客文件

  2. 以 ssh 的方式将博客站点的存储库拉取到本地，将写好的 .md 源文件拷贝到本地博客站点存储库中

  3. 以 git 的方式形成提交，push 到远端，自动触发远端博客站点存储库的 githubActions

  4. githubActions 自动对博客源码进行编译部署

注意：虽然本博客支持云端写作触发自动化部署，但是不推荐这么做，因为这样会导致在本地某磁盘某文件夹下编写的 .md 博客文件滞后于远端博客站点存储库

前置工作

首先，自行在本地配置好 nodejs 环境，不会就=>百度

然后，确保你在本地和云端之间配置好了 ssh 加密通信，不会可以参考我的这篇博客

第一步，本地生成博客站点的基本代码并预览（以我的 github 账户名 brannua 为例）

# 本地生成博客站点的基本代码
sudo npm install hexo-cli -g
hexo init blog
cd blog

# 预览博客
hexo s

# 终止预览：ctrl+c

第二步，更换博客主题

主题列表：https://hexo.io/themes/

  • 本教程选用：hexo-theme-yun
git clone https://github.com/YunYouJun/hexo-theme-yun ./themes/yun --depth 1

cd ./themes/yun

rm -rf ./.git ./.github

cd ../../

npm install hexo-render-pug --save

vim _config.yml

将 theme: landscape 修改为 theme: yun 后保存退出

删掉 ./_config.landscape.yml

删掉 package.json 中的 "hexo-theme-landscape": "^0.0.3"

删掉 yarn.lock 中与 landscape 相关的那一段

# 再次预览博客就可看到使用了新主题
hexo s

第三步，自定义配置博客主题

首先安装依赖

# RSS 相关
npm i hexo-generator-feed --save

# 文章字数统计和阅读时长估计相关
npm i hexo-wordcount --save

# 本地搜索相关
npm i hexo-generator-search --save

# 站点部署相关
npm i hexo-deployer-git --save

# dev 热更新
npm i -g browser-sync
npm i hexo-browsersync -D

然后參考hexo-theme-yun在 source/_data/yun.yml 中进行博客站点的自定义配置即可

第四步，站点远端部署

  • 首先，在 github 上创建两个仓库

    • blog（privacy 备份站点源码的仓库）

    • brannua.github.io（public 站点的部署仓库）

  • 然后，编辑站点源码根目录下的配置文件 _config.yml，写入以下内容保存退出
deploy:
  type: git
  repo: git@github.com:Brannua/brannua.github.io.git
  branch: master
  message: "deploy 自动推送: 站点更新"
  • 接下来生成并部署静态站点到 github 仓库 brannua.github.io，默认 master 分支（部署会需要一点时间）
  • 预览地址：https://brannua.github.io
hexo clean		# 清除旧的静态站点
hexo generate	# 生成新的静态站点 /public
hexo deploy   # 部署

第五步，实现站点的自动部署

  • 首先将站点源码推送到 github 仓库 blog，默认 master 分支
git init
git remote add origin [sshRepoLink]
git pull 
git add .
git commit -m "the first commit."
git push origin master
  • 然后打通云端的两仓库（源码仓库和部署库使用 ssh 加密通信）
# 首先在本地生成非对称加解密密钥对
cd ~/.ssh
ssh-keygen -t rsa -f gh-hexo-deploy -C [邮箱]

# 给源码仓库配置私钥：https://github.com/Brannua/blog -> Settings -> Secrets -> New repository secret，填入以下配置后点击 Add secret
Name: HEXO_DEPLOY_KEY
Value: [cat ~\.ssh\gh-hexo-deploy]

# 给部署仓库配置公钥：https://github.com/Brannua/brannua.github.io -> Settings -> Deploy keys -> Add deploy key，填入以下配置并勾选 Allow write access 然后点击 Add key 
Title: HEXO_DEPLOY_PUB
Value: [cat ~\.ssh\gh-hexo-deploy.pub]
  • 最终实现站点的自动部署
# 给站点源码编写自动部署的配置文件
cd blog/.github
mkdir workflows
cd workflows
vim deploy.yml

# 写入如下配置，那么从此以后，只需维护源码仓库，而无需手动部署站点
name: Hexo Deploy

on:
  push:
    branches:
      - master

jobs:
  build:
    runs-on: ubuntu-18.04
    if: github.event.repository.owner.id == github.event.sender.id

    steps:
      - name: 1. Setup Node.js...
        uses: actions/setup-node@v1
        with:
          node-version: '12.x'

      - name: 2. Setup Hexo...
        run: |
          npm i hexo-cli -g

      - name: 3. Setup Git...
        run: |
          git config --global user.name "Brannua"
          git config --global user.email "374453156@qq.com"

      - name: 4. Setup ssh...
        env:
          ACTION_DEPLOY_KEY: ${{ secrets.HEXO_DEPLOY_KEY }}
        run: |
          mkdir ~/.ssh/
          echo "$ACTION_DEPLOY_KEY" > ~/.ssh/id_rsa
          sudo chmod 700 ~/.ssh ~/.ssh/id_rsa
          sudo ssh-keyscan github.com >> ~/.ssh/known_hosts

      - name: 5. Checkout source...
        uses: actions/checkout@v2
        with:
          ref: master

      - name: 6. Install dependences...
        run: |
          npm install

      - name: 7. Build and Deploy...
        run: |
          hexo clean
          hexo generate
          hexo deploy

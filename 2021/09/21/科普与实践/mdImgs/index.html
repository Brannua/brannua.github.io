笔者的工作环境

关于图床

  • 个人观点和需求
    • 自己应对自己的资源具有绝对掌控权
    • 选择的服务提供商体量应足够大，能够提供长期稳定的服务
    • 数据隐私性不考虑因为只是上传个人博客的图片资源，只希望数据安全健在不被和谐，上传下载不被限速（别瞅了某度某盘说的就是你）
  • 关于第三方图床平台
    • 在我的认知范围内，没有发现完全满足我个人需求的平台，但是有个比较知名的平台叫路过图床
  • 我的选择
    • github？满足我的需求，但是由于 gfw 的存在，其网速不理想
    • 所以我选择了国内的替代品gitee！注意：gitee 图床只支持不大于 1 MB 的图片，一般截图 size 较小（可以），而相机拍摄的原图 size 较大（不行）

关于 gitee 如何获取图片链接

  1. 通过 git 的方式将你的图片上传到你 gitee 帐号的仓库中
  2. 查看图片：https://gitee.com/pj-l/imgs-1/blob/master/demo.png
  3. 获取链接：https://gitee.com/pj-l/imgs-1/raw/master/demo.png（将 blob 替换为 raw）

接下来实现通过 picGo 往 gitee 传图

  • 首先在 gitee 个人帐号的设置中生成私人令牌（token），并新建开源存储库作为图床（需要勾选初始化仓库，建议设置开源许可证，如 MIT）

    • 私人令牌只需勾选 projects
    • gitee 单个存储库有容量限制，存满就建立新仓库，仓库命名方式可以为 imgs-n，n 替换为 1 2 3 4 5 …… 以此类推

  • 在本地安装 node.js 环境，自行=>百度

  • 然后使用图形化客户端 picGo：github|doc，你给它配个目标地址（你的图床），加上token（私人令牌），就能发请求传图，回调获取在线图片地址

    • linux 用户使用 AppImage 的方式，即：下载一个可执行文件，赋予可执行权限，双击执行无需安装无需考虑依赖问题
    • 双击 picGo 的 .AppImage file 执行

    • 系统托盘右键其图标，点击打开详细窗口；
    • 由于 picGo 自带的 uploader 没有 gitee 的，所以我们需要下个插件叫 gitee-uploader，点击插件设置，搜索 gitee，安装 gitee-uploader

    • 然后点开图床设置，点击 gitee，填入用户名/仓库名（eg：pj-l/imgs-1） 和 gitee 私人令牌（token），branch 默认 master，其余可以不填
    • 点击确定和设为默认图床，你就已经可以使用 picGo 往 gitee 上传图了

接下来给 typora 配置自动上传

给 .AppImage file 赋予 755 权限，让 typora 调用它即可；可参考：https://github.com/Molunerfinn/PicGo/issues/513

至此，你就可以直接截图，将截图复制到剪贴板，直接粘贴进 typora md 文件中，实现自动上传至 gitee，并自动将图片本地路径转换为在线链接了

并且，上传高效，服务稳定，管理方便，数据安全。
